 ۱. زیرمسئله: مدیریت حالت‌های چرخه حیات تیکت
شرح مسئله
هر تیکت در طول عمر خود پنج حالت متوالی را طی می‌کند:

NEW (ایجاد شده)

ASSIGNED (ارجاع داده شده)

IN_PROGRESS (در حال رسیدگی)

RESOLVED (حل شده)

CLOSED (بسته شده)

در کد فعلی، این حالت‌ها به صورت رشته‌ای مدیریت شده و رفتار هر حالت با استفاده از زنجیره‌ای از دستورات if-else پیاده‌سازی شده است.
